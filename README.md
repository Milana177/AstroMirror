# 🔮 AstroMirror — Астрологическое зеркало

## 📌 Описание проекта

**AstroMirror** — это веб-приложение для астрологических практик и самопознания. Пользователи могут строить натальные карты, рассчитывать матрицу судьбы на основе 22 арканов, получать ежедневные предсказания Таро и делать расклады "Три карты" с получением развернутых ответов на волнующие вопросы. Приложение сохраняет историю всех расчетов и предоставляет доступ к библиотеке старших арканов Таро. Все данные хранятся локально на устройстве пользователя, что обеспечивает полную конфиденциальность.

## 🛠 Технологический стек

### 🖥 Frontend

| Технология | Обоснование |
|------------|-------------|
| **HTML5** | Семантическая разметка для структуры приложения, обеспечивает доступность и SEO |
| **CSS3** | Кастомные стили с анимациями (звезды, градиенты, свечение), адаптивный дизайн под мобильные устройства |
| **Vanilla JavaScript (ES6+)** | Приложение не требует сложной реактивности в реальном времени. Астрологические расчеты выполняются на клиенте, данные хранятся в localStorage. Отсутствие фреймворков упрощает разработку, снижает порог входа и обеспечивает максимальную производительность без лишних зависимостей. |
| **MySQL** | Для хранения данных пользователей, истории расчетов, карты дня. |

**Почему выбран чистый JS вместо фреймворков:**
- Приложение имеет умеренную сложность (нет требований к real-time обновлениям)
- Все расчеты (матрица судьбы, выбор карт) выполняются синхронно на клиенте
- MySQL обеспечивает достаточное хранилище для пользовательских данных
- Отсутствие бэкенда упрощает деплой и снижает затраты на хостинг

### ⚙ Backend

| Технология | Обоснование |
|------------|-------------|
| **Отсутствует (клиент-сайд архитектура)** | На текущем этапе все расчеты выполняются на стороне клиента. Это обеспечивает полную конфиденциальность данных пользователя (все хранится локально) и позволяет использовать приложение без интернета после загрузки. |

### 🗄 База данных

| Технология | Обоснование |
|------------|-------------|
| **MySQL** | Используется для хранения: пользователей (`astroUsers`), текущей сессии (`astroUser`), истории расчетов (`astroHistory_{userId}`), карты дня (`cardOfDay_{userId}`). |

### 🌐 Инфраструктура

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Frontend хостинг** | GitHub Pages | Бесплатный хостинг для статического сайта, простая публикация из репозитория, автоматическое обновление при изменении кода |
| **Система контроля версий** | Git + GitHub | Для хранения кода и отслеживания изменений |

## 🎨 Визуальное проектирование интерфейса

### 1. Главная страница (`/`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Шапка:** Логотип "AstroMirror", навигация (Натальная карта, Матрица судьбы, Таро, Вопрос в колоду, История), меню пользователя (имя + кнопка "Выйти") или кнопка "Войти"
2. **Hero-блок:** Заголовок "Зеркало твоей души", подзаголовок "Открой свои истинные пути через звезды, числа и древние символы"
3. **Секция «Натальная карта»:** Карточка с иконкой 🌙, заголовком "Астрологический анализ", описанием и кнопкой "Перейти"
4. **Секция «Матрица судьбы»:** Карточка с иконкой 🔮, заголовком "Расчет Матрицы", описанием и кнопкой "Перейти"
5. **Секция «Таро»:** Две карточки: "Карта дня" и "Библиотека Таро"
6. **Секция «Вопрос в колоду»:** Карточка для расклада на трех картах
7. **Футер:** Информация о проекте, контакты, ссылка на политику конфиденциальности

**Динамический контент:** Отображение имени пользователя в шапке после авторизации

## 🗄 Схема данных и API

### Основные модели данных

#### 1. Модель пользователя (User)

| Поле | Тип | Описание |
|------|-----|----------|
| id | number | Уникальный идентификатор (Date.now()) |
| name | string | Имя пользователя |
| email | string | Электронная почта (уникальная) |
| password | string | Пароль (хранится в открытом виде для MVP) |

**Хранилище:** `localStorage.astroUsers`

---

#### 2. Модель записи истории (HistoryEntry)

| Поле | Тип | Описание |
|------|-----|----------|
| type | string | Тип записи: 'matrix', 'cardOfDay', 'majorArcana' |
| data | object | Данные расчета в JSON формате |
| timestamp | string | ISO-строка даты и времени |

**Хранилище:** `localStorage.astroHistory_{userId}`

**Пример data для matrix:**
```json
{
  "name": "Анна",
  "date": "1990-05-15",
  "birthArcane": 7,
  "destinyArcane": 8,
  "talentArcane": 3
}
